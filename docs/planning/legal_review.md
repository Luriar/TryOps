## 1. 개인정보보호법 — 영상정보처리기기 규정 (제25조)

### 1.1 법령 조항 (🟢 직접 인용)

**개인정보보호법 제25조 제2항**:

> *"누구든지 불특정 다수가 이용하는 목욕실, 화장실, 발한실, 탈의실 등 개인의 사생활을 현저히 침해할 우려가 있는 장소의 내부를 볼 수 있도록 고정형 영상정보처리기기를 설치·운영하여서는 아니 된다. 다만, 교도소, 정신보건 시설 등 법령에 근거하여 사람을 구금하거나 보호하는 시설로서 대통령령으로 정하는 시설에 대하여는 그러하지 아니하다."*

출처: 국가법령정보센터, 개인정보 보호법 (법률 제20897호, 2025.07.22)
링크: https://www.law.go.kr/법령/개인정보보호법

### 1.2 TryOps 적용 분석 (🟢)

**영상정보처리기기 정의**:
- 일정한 공간에 지속적으로 설치되어 사람 또는 사물의 영상을 촬영하거나 이를 유·무선망을 통하여 전송하는 장치
- **핵심 키워드: "영상을 촬영"**

**mmWave·WiFi CSI 센서의 분류**:
- 24GHz 레이더 반사 및 WiFi 신호의 채널 상태 정보(Channel State Information) 변화를 측정
- **영상 촬영 안 함** — 광학 센서·이미지 센서 없음
- → 영상정보처리기기 **해당 안 됨**

**법적 결론 🟢**: TryOps의 mmWave·WiFi CSI 센서는 개인정보보호법 제25조의 영상정보처리기기 규제에서 **면제**. 피팅룸·탈의실 설치 가능. **이게 TryOps의 결정적 법적 우위**.

### 1.3 함의 — 시장 진입의 법적 우위

피팅룸은 *"탈의실"* 에 해당하므로 CCTV·영상 카메라 설치 불가. 영상 기반 매장 분석 솔루션(RetailNext·데이터넥스트·인사이트인 등)은 피팅룸 내부 데이터 자체를 확보 불가. **TryOps는 법적으로 유일하게 피팅룸 내부 데이터를 합법적으로 수집할 수 있는 솔루션**.

## 2. 개인정보보호법 — 일반 규정 (제2조·제15조·제17조)

### 2.1 개인정보 정의 (🟢)

**제2조 제1호**:
> *"개인정보란 살아 있는 개인에 관한 정보로서 다음 각 목의 어느 하나에 해당하는 정보를 말한다. 가. 성명, 주민등록번호 및 영상 등을 통하여 개인을 알아볼 수 있는 정보. 나. 해당 정보만으로는 특정 개인을 알아볼 수 없더라도 다른 정보와 쉽게 결합하여 알아볼 수 있는 정보"*

### 2.2 TryOps 데이터의 개인정보 해당 여부 (🟡)

**원천 CSI 신호 (단독)**:
- WiFi 신호 변화 데이터 자체로는 개인 식별 불가
- *"개인정보"* 해당 안 됨

**Try-on session 단위 데이터**:
- RFID 이벤트 + POS 결제 시각 + CSI 윈도우 조합
- *"다른 정보와 쉽게 결합"* 가능성 — POS의 카드번호와 결합 시 개인 식별 가능
- 🟡 회색지대 — *"가명정보"* 또는 *"개인정보"* 분류 가능성

**대응 (안전 측 설계)**:
- POS 데이터의 카드번호·전화번호 등은 매장 단위 익명 집계만 사용
- 본사 전송 데이터는 SKU 단위 + 시간 단위 집계만
- session 단위 데이터는 7~30일 후 자동 삭제
- 매장 raw CSI는 7일 후 자동 삭제

### 2.3 정보주체 통지 의무 (제15조·제17조)

**🟢 통지 의무 조항**:
- 개인정보 수집 시 정보주체(고객)에게 *"수집·이용 목적"*, *"수집 항목"*, *"보유 기간"*, *"동의 거부 권리"* 고지
- 영상정보처리기기 외에는 안내판 의무 없음 (제25조 해당 아닐 시)

**TryOps 대응**:
- 매장 입구 안내문 필수 (법적 의무는 아니지만 신뢰 설계로 권장)
- 매장 직원 5개 표준 질문 답변 매뉴얼 (`product_strategy.md` 섹션 7.1 참조)
- 본사 개인정보처리방침에 *"매장 비영상 센싱"* 명시

### 2.4 정보주체의 권리 (제35조·제36조·제37조)

**🟢 법적 권리**:
- 열람 요구권 (제35조)
- 정정·삭제 요구권 (제36조)
- 처리정지 요구권 (제37조)

**TryOps 대응의 어려움**:
- *"개인 식별 안 되는 익명 데이터"* 의 경우 본인 데이터 특정이 불가능
- 손님이 *"내 데이터 빼달라"* 요청 시 *"매장 단위 익명이라 개인 식별이 안 됩니다"* 답변
- 그러나 *"매장 출입 거부"* 또는 *"매장 전체 도입 거부"* 권리는 매장 차원의 해결 사항

🟡 회색지대: PoC 단계 손님이 *"내 데이터 처리정지"* 명시 요구 시 처리 절차 미정의. 법무 자문 필요.

## 3. 개인정보보호법 — 가명정보·익명정보 (제28조의2 외)

### 3.1 가명정보 vs 익명정보 정의 (🟢)

- **가명정보**: 개인정보의 일부를 삭제하거나 일부 또는 전부를 대체하는 등의 방법으로 추가 정보 없이는 특정 개인을 알아볼 수 없도록 처리한 정보 (제2조 제1호의 다)
- **익명정보**: 시간·비용·기술 등을 합리적으로 고려할 때 다른 정보를 사용하여도 더 이상 개인을 알아볼 수 없는 정보 (제58조의2)

### 3.2 TryOps 데이터 분류 (🟡)

| 데이터 유형 | 분류 | 사유 |
|---|---|---|
| 원천 CSI 신호 (단독) | 🟢 익명정보 | 개인 식별 불가능 |
| Try-on session 마트 (집계) | 🟢 익명정보 | SKU·시간·매장 단위 집계 |
| RFID + POS + CSI 통합 (raw) | 🟡 가명정보 가능성 | 추가 정보 결합 시 개인 식별 가능 |
| 본사 전송 데이터 | 🟢 익명정보 | 의도적 익명 집계만 전송 |

**가명정보 처리 시 의무 (제28조의5)**:
- 안전성 확보 조치 (기술적·관리적)
- 가명 처리 기록 보관
- 통계작성·과학적 연구·공익적 기록 보존 외 목적 사용 금지

### 3.3 안전 측 설계

TryOps는 *"raw 데이터는 매장 내 가명정보, 본사 전송은 익명정보"* 의 이중 구조로 설계. 본사가 받는 데이터는 익명정보로 분류돼 가명정보 처리 의무 미해당.

## 4. 근로자 감시 설비 — 근로자참여 및 협력증진법

### 4.1 법령 조항 (🟢)

**근로자참여 및 협력증진법 제20조 제14호**:

> *"노사협의회는 다음 각 호의 사항을 협의한다. 14. 사업장 내 근로자 감시 설비의 설치"*

출처: 국가법령정보센터, 근로자참여 및 협력증진법
링크: https://www.law.go.kr/법령/근로자참여및협력증진법

**적용 대상**:
- 상시 30명 이상 근로자를 사용하는 사업장 (제4조)
- 30명 미만은 노사협의회 의무 없음

### 4.2 TryOps의 근로자 감시 설비 해당 여부 (🟡)

**근로자 감시 설비의 정의**:
- 법령에 명확한 정의 없음
- 실무적으로 *"근로자의 업무 수행 또는 작업 행위를 지속적으로 감시할 수 있는 설비"* 로 해석

**TryOps 분석**:
- 매장 직원의 *"위치·동선·체류"* 가 CSI 신호로 잡힐 가능성 → 🟡 회색지대
- 매장 손님과 직원을 구분할 정확한 방법 없음
- *"직원별 평가 지표 미제공"* 의 제품 정책으로 우회 가능
- 단, 노사협의회 협의 대상으로 해석될 가능성 있음

### 4.3 대응

**대형 매장 (30명 이상 사업장)**:
- 도입 전 노사협의회 공식 협의
- 협의 결과 문서화
- 직원 동의서 (수집 정보, 목적, 보관 기간 명시)
- 본인 데이터 열람 권리 보장

**소형 매장 (30명 미만)**:
- 노사협의회 의무 없음
- 단, 직원 설명회 + 동의서는 권장 (분쟁 예방)

### 4.4 제품 정책 — *"감시"가 아닌 *"운영"* 프레이밍 (🟡)

| 위험한 표현 | 안전한 표현 |
|---|---|
| "직원 응대 평가" | "매장 운영 데이터" |
| "직원 개입 미참여 알람" | "피팅룸 대기 시간 알람" |
| "직원별 KPI 측정" | "매장 KPI 측정" |
| "실시간 감시" | "실시간 운영 모니터링" |

법적 안전성 + 직원 우호도 양면에 핵심.

## 5. 의료기기법 — 가장 위험한 법령 영역

### 5.1 법령 조항 (🟢)

**의료기기법 제2조**:

> *"의료기기란 사람이나 동물에게 단독 또는 조합하여 사용되는 기구·기계·장치·재료·소프트웨어 또는 이와 유사한 제품으로서 다음 각 호의 어느 하나에 해당하는 제품을 말한다.*
> *1. 질병을 진단·치료·경감·처치 또는 예방할 목적으로 사용되는 제품*
> *2. 상해 또는 장애를 진단·치료·경감 또는 보정할 목적으로 사용되는 제품*
> *3. 구조 또는 기능을 검사·대체 또는 변형할 목적으로 사용되는 제품*
> *4. 임신을 조절할 목적으로 사용되는 제품"*

출처: 국가법령정보센터, 의료기기법
링크: https://www.law.go.kr/법령/의료기기법

### 5.2 TryOps의 의료기기 해당 위험 (🟡)

**RF 센싱의 호흡·심박 측정 가능성**:
- mmWave·CSI 기반 호흡 감지는 연구 수준에서 기술적으로 가능 (v2 아키텍처는 해당 기능 미구현·미수집)
- *"호흡 측정"*, *"심박 측정"*, *"응급 감지"* 마케팅 시 의료기기 분류 가능성
- 의료기기로 분류되면 식약처 인증 필수, 미인증 시 형사 처벌 가능

### 5.3 회피 전략 (🟢 결정적 안전 설계)

**Phase 1·2 (PoC·Pilot 단계)**:
- 호흡·심박을 **숫자로 표시 안 함**
- *"활동 강도 점수"* 또는 *"행동 강도 지표"* 로 우회 표기
- 의료적 주장 (응급 감지·건강 모니터링·진단·치료) **절대 금지**

**Phase 3 이후**:
- 호흡·심박 데이터 활용 시 식약처 사전 검토
- 별도 *"의료기기 제품 라인"* 분리 검토 (가정용 헬스케어 등)

**필수 명시 문구**:
> *"본 서비스는 의료기기가 아니며 의료적 진단·치료 목적으로 사용할 수 없습니다."*

이 문구를 매장 안내문·본사 계약서·웹사이트에 명시.

### 5.4 마케팅·홍보 가이드

**위험 표현 (절대 금지)**:
- "응급 상황 감지"
- "건강 모니터링"
- "호흡 이상 알림"
- "심박 측정"
- "노인 안전 감지"

**안전 표현 (대체)**:
- "장시간 무활동 알림"
- "피팅룸 회전율 분석"
- "활동 강도 변화 감지"
- "매장 운영 효율 분석"

## 6. 위치정보법 — 비해당 검증

### 6.1 법령 조항 (🟢)

**위치정보의 보호 및 이용 등에 관한 법률 제2조**:

> *"위치정보란 이동성이 있는 물건 또는 개인이 특정한 시간에 존재하거나 존재하였던 장소에 관한 정보로서 「전기통신사업법」 제2조 제2호 및 제3호에 따른 전기통신설비 및 전기통신회선설비를 이용하여 수집된 것을 말한다."*

핵심 키워드: **"전기통신설비·전기통신회선설비 이용 수집"**.

### 6.2 TryOps 적용 분석 (🟢)

**mmWave·WiFi CSI 센서**:
- 레이더 반사 및 WiFi 신호의 공간 변화 측정 (반사·반향·간섭 패턴)
- 스마트폰 등 *"이동성 물건의 위치"* 를 측정하지 않음
- 전기통신설비 이용한 위치 수집 **해당 안 됨**

**법적 결론 🟢**: 위치정보법 **적용 대상 아님**.

### 6.3 금지선 — 향후 MAC 트래킹 절대 금지

**위험 시나리오**:
- 매장 WiFi 접속 시 스마트폰 MAC 주소 수집
- 손님 동선을 MAC 주소로 추적

**이 기능은 위치정보법 적용 대상이 되며**:
- 위치정보사업자 허가 필수 (방통위)
- 정보주체 명시적 동의 필수
- 본인 위치정보 확인·삭제 요구권 보장
- 미준수 시 5년 이하 징역 또는 5천만원 이하 벌금 (제40조)

**TryOps 제품 정책 (🟢 안전 설계)**:
- **MAC 주소 수집·저장·분석 절대 금지**
- 향후 *"손님 식별"* 기능 추가 검토 시에도 MAC 대신 익명 ID 사용
- 매장 게이트웨이는 WiFi 신호만 수집, 접속 기기 정보 미수집

## 7. 전파법 / KC 인증

### 7.1 법령 요지 (🟢)

**전파법 제58조의2 (적합성평가)**:
- 전파를 사용하는 기자재는 적합성평가(KC 인증) 필수
- 미인증 기기 판매·유통 시 형사 처벌 가능

### 7.2 TryOps 적용 (🟡)

**ESP32-C6 모듈 자체**:
- 글로벌 인증 (FCC·CE·KC) 보유 모듈 소싱 가능
- 칩 단독 사용은 인증 추가 불요

**mmWave 모듈 (HLK-LD2410C/LD2450) 🔴 신규 확인 필요**:
- 제조사 확인 기준 FCC·CE 인증만 확인됨, **KC 인증 미확인**
- Stage 0(자체 실험)은 문제 없으나 Stage 1 매장 설치 전 KC 기인증 모듈 소싱 또는 자체 적합성평가 필요

**TryOps 노드·게이트웨이 (자체 제품)**:
- 하우징·펌웨어 조합 시 KC 인증 추가 필요 (🟡 회색지대)
- 단순 *"내부 사용"* 은 인증 면제 가능
- *"판매·유통"* 시 KC 인증 필수

### 7.3 단계별 인증 계획

| 단계 | 인증 상태 | 사유 |
|---|---|---|
| Phase 1·2 (PoC) | 면제 가능 | 자체 사용 목적, 판매·유통 안 함 |
| Phase 3 (상용화) | KC 인증 필수 | 매장에 노드 판매·임대 |
| Phase 4 (글로벌) | FCC·CE 추가 | 미국·유럽 매장 확장 시 |

KC 인증 비용: 약 500~1,500만원, 기간 3~6개월 (🟡 일반 추정).

  **TryOps 선제 대응(설계 원칙으로 명문화)**: ① raw CSI를 게이트웨이 밖으로 전송·보존하지 않음

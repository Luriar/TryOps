# TryOps Simulator Specification

> 본 문서는 Phase 1 시뮬레이션 데이터 생성기의 명세 골격이다. 실제 매장 데이터 없이 알고리즘·UI·분석 마트를 검증하기 위한 합성 데이터 환경.
>
> **본 문서의 위치**: PoC 1단계 진입 전 시뮬레이션 단계에서 활용. 실제 매장 설치 후 v2로 정밀화.

---

## 0. 시뮬레이터의 목적

본 시뮬레이터가 답해야 하는 질문:

1. **알고리즘 검증**: Joint Signal 5종 알고리즘이 의도한 시나리오에서 의도한 결과를 내는가?
2. **UI 시연**: 본사·매장 대시보드가 실제 비즈니스 의사결정에 도움되는가?
3. **데이터 마트 검증**: ETL 파이프라인이 100매장 규모 데이터에서 안정적으로 동작하는가?
4. **영업 데모**: 본사 영업 시 시뮬레이션 데이터로 가치 시연 가능한가?

## 1. 시뮬레이터 구성

### 1.1 출력 데이터 3종

```
[Simulator Output]
├── rfid_events.parquet      (RFID 입출고 이벤트)
├── pos_transactions.parquet (POS 결제 이벤트)
└── csi_windows.parquet      (CSI 활동 윈도우)
```

각 파일은 일 단위 분할. 100매장 × 30일 = 3,000개 파일.

### 1.2 매장 합성 (100매장)

| 매장 카테고리 | 매장 수 | 특성 |
|---|---|---|
| 애슬레저 강세 매장 | 30 | 피팅룸 의존도 매우 높음, try-on 일평균 150건 |
| 언더웨어 매장 | 20 | 피팅룸 의존도 중간, try-on 일평균 80건 |
| 컨템포러리 매장 | 25 | 사이즈 단순, try-on 일평균 100건 |
| 다양 카테고리 매장 | 15 | 평균 패턴 |
| 피크 매장 | 10 | 주말 매출 집중, try-on 일평균 250건 |

매장당 다음 메타 설정:
- 피팅룸 개수: 4~8개
- 평일 영업: 10~22시
- 주말 영업: 10~22시 (피크 효과 적용)

### 1.3 SKU 합성 (500개)

| 카테고리 | SKU 수 | 사이즈 | 색상 |
|---|---|---|---|
| 레깅스 | 80 | 3~5종 (S, M, L, XL 등) | 4~6종 |
| 스포츠 브라 | 60 | 3~4종 | 3~5종 |
| 후드 | 70 | 3~5종 | 4~6종 |
| 셔츠 | 90 | 3~5종 | 5~7종 |
| 원피스 | 50 | 3~4종 | 4~6종 |
| 언더웨어 | 80 | 3~4종 | 3~5종 |
| 기타 | 70 | 다양 | 다양 |

SKU별 가격 분포: 3만원 ~ 15만원 (애슬레저 평균 7만원).

## 2. 의도적 시나리오 주입

본 시뮬레이터의 핵심 가치는 *"의도된 케이스 발견"* 의 검증 가능성. 다음 시나리오를 의도적으로 주입.

### 2.1 시나리오 1: 사이즈 차트 불일치

**의도**: 특정 SKU의 사이즈 차트와 실제 핏 불일치 → high try-on / low conversion

| 변수 | 값 |
|---|---|
| 대상 SKU | "SKU_LEGGINGS_M_BLACK" 등 5개 |
| 의도 패턴 | M 입실 → 5분 후 L 추가 입실 → 7분 체류 → 미구매 |
| Hesitation Score 기대값 | 0.7+ |
| 매장 적용 | 100매장 중 70% |

### 2.2 시나리오 2: 가격 vs 핏 분리 가능 케이스

**의도**: 가격 인하 의사결정 회의에서 *"가격 문제 아니라 핏 문제"* 분리

| 변수 | 값 |
|---|---|
| 대상 SKU | 5개 |
| 의도 패턴 | 단일 사이즈 7분+ 체류 → 미구매 (가격 인하해도 안 살 SKU) |
| Hesitation Score 기대값 | 0.6~0.8 |

### 2.3 시나리오 3: 매장 운영 병목

**의도**: 피크 시간대 피팅룸 부족으로 전환 하락

| 변수 | 값 |
|---|---|
| 적용 매장 | 피크 매장 10개 |
| 시간대 | 토요일 15~18시 |
| 의도 패턴 | 매장 외부 CSI 활동 강도 0.8+ + Fitting Friction Score 0.7+ |
| Friction Score 기대값 | 0.7+ |

### 2.4 시나리오 4: Phantom Try-on

**의도**: RFID 누락 케이스 발견

| 변수 | 값 |
|---|---|
| 의도 비율 | 전체 try-on의 5% |
| 패턴 | CSI 점유 6분+ + RFID enter 이벤트 0 |
| Phantom Detection 기대 정확도 | 80%+ |

### 2.5 시나리오 5: Companion Effect

**의도**: 가족·친구 동행 케이스

| 변수 | 값 |
|---|---|
| 의도 비율 | 전체 try-on의 10% |
| 패턴 | CSI 점유 추정 2+ + 동시 다른 SKU 입실 |
| Companion Probability 기대값 | 0.6+ |

## 4. 시뮬레이터 검증 기준

본 시뮬레이터의 *"진짜 가치"* 는 알고리즘이 의도된 시나리오를 얼마나 잘 발견하는가.

| 검증 항목 | 목표 정확도 |
|---|---|
| 시나리오 1 (사이즈 차트 불일치) Hesitation 발견 | 90%+ |
| 시나리오 2 (가격 vs 핏 분리) 정확도 | 80%+ |
| 시나리오 3 (매장 운영 병목) Friction 발견 | 90%+ |
| 시나리오 4 (Phantom Try-on) 정확도 | 80%+ |
| 시나리오 5 (Companion) 정확도 | 70%+ |

이 5가지가 80%+ 도달 시 알고리즘 본 PoC 준비 완료.

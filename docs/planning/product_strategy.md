# TryOps Product Strategy

## 카메라 없는 피팅룸 인텔리전스 플랫폼

---

## 0. Executive Summary

TryOps는 RFID와 POS 데이터가 설명하지 못하는 피팅룸 내부의 행동 공백을 mmWave·WiFi CSI 센싱 신호로 보완해, K-D2C 브랜드 본사가 상품·사이즈·매장 운영 의사결정을 더 정확하게 하도록 돕는 오프라인 리테일 전환 인텔리전스 SaaS다.

핵심 명제:
- RFID는 **무엇이 들어갔는지**를 본다
- POS는 **결국 무엇이 팔렸는지**를 본다
- mmWave·WiFi CSI 센서는 그 사이의 **망설임·반복 비교·체류 강도·대기 마찰·동행 가능성**을 본다

TryOps는 RFID를 대체하지 않는다. RFID의 가치를 확장한다. RFID-only 솔루션이 *"상품이 들어갔다가 안 팔렸다"* 까지 말할 수 있다면, TryOps는 *"그 상품이 왜 안 팔렸을 가능성이 있는지"* 에 대한 근거 후보를 만든다.

**1차 타겟**: K-D2C 패션 브랜드 본사 (직영점 10~50개, RFID 도입 또는 검토 중)
**1차 카테고리**: 언더웨어·스포츠웨어·애슬레저
**가격**: Funnel Module 매장당 월 15만~30만원, Flow Module 매장당 월 5만~10만원
**MVP 핵심 가설**: Hesitation Score가 RFID-only funnel보다 high try-on / low conversion 상품의 원인 후보를 더 잘 구분할 수 있다

## 1. 제품 한 줄 소개

**TryOps는 mmWave·WiFi CSI 센서, RFID 상품 이벤트(EPC), POS 결제 데이터를 시간축으로 결합해 K-D2C 브랜드 본사가 '고객이 입어봤지만 사지 않은 이유'를 추정하고, 상품·사이즈·직원 응대·피팅룸 운영 개선 액션으로 연결할 수 있게 하는 피팅룸 행동 인텔리전스 SaaS다.**

---

## 2. 문제 정의

오프라인 의류 매장에서 피팅룸은 구매 전환 직전의 핵심 구간이다. 고객이 상품을 들고 피팅룸에 들어갔다는 것은 단순 방문보다 구매 의도가 훨씬 강한 상태다. 그러나 브랜드 본사는 이 구간에서 가장 중요한 질문에 답하지 못한다.

- 어떤 상품은 많이 입어봤지만 왜 구매되지 않았는가?
- 사이즈 문제인가, 핏 문제인가, 소재 문제인가, 가격 문제인가?
- 피팅룸 대기나 직원 응대 부족 때문에 이탈했는가?
- 매장별 전환 차이가 상품 문제인지, 직원 응대 문제인지, 피팅룸 병목 문제인지 구분할 수 있는가?

기존 데이터의 한계:

| 데이터 | 알 수 있는 것 | 모르는 것 |
|---|---|---|
| POS | 무엇이 팔렸는가 | 무엇이 안 팔렸고, 왜 안 팔렸는가 |
| RFID | 어떤 SKU·사이즈·색상이 피팅룸에 들어가고 나왔는가 | 피팅룸 안에서 고객이 무엇을 느꼈는가 |
| CCTV | 매장 일반 동선, 입구, 계산대 등 | 피팅룸 내부 행동 (법적·사회적 영상 금지) |
| 직원 관찰 | 현장 감각, 고객 응대 경험 | 본사 단위로 축적 가능한 정량 데이터 |
| 리뷰/반품/CS | 구매 이후 불만 | 구매 전 실패 신호 |

핵심 문제:

> **피팅룸 내부 행동 데이터 공백 때문에, 본사는 try-on 이후 구매 실패 원인을 구조적으로 알 수 없다.**

## 3. Why Now

TryOps가 지금 의미 있는 이유는 다음 다섯 가지다.

**첫째, K-패션의 글로벌 확장 변곡점.** 2024~2026년 사이 K-D2C 브랜드의 글로벌 매장 확장이 가속화되고 있다. 마뗑킴·마르디 메크르디·안다르·젝시믹스 등이 동남아·미국·일본에 직영점을 늘리고 있고, 본사 입장에서 매장 운영을 데이터로 표준화·통제하려는 수요가 강해졌다.

**둘째, 본사 데이터팀의 보편화.** 5년 전엔 중견 브랜드에 거의 없던 데이터팀이 이제 표준 조직이다. 이들은 *"데이터로 의사결정을 바꿔야 한다"* 는 미션을 갖고 있지만, 오프라인 피팅룸은 여전히 갖춘 도구가 없다.

**셋째, 개인정보 규제 강화.** 2023년 개정 개인정보보호법 이후 영상정보처리기기 규제가 강화됐다. 피팅룸 내부 영상은 명시적으로 금지되고, 매장 영상의 본사 전송도 제약이 늘었다. 비영상 센싱의 법적 정합성이 강점이 된다.

**넷째, RFID 도입 단계 성숙.** 대형 SPA가 깔아둔 인프라를 중견 D2C도 따라가는 단계. *"RFID 도입했는데 활용을 못 한다"* 가 본사 데이터팀의 흔한 고민이 됐고, TryOps는 그 위에 얹는 분석 레이어로 정확히 자리 잡는다.

**다섯째, RF sensing 기술의 표준화·오픈소스화.** IEEE 802.11bf WiFi 센싱 표준이 2024년 말 비준되었고, Espressif esp_wifi_sensing 등 칩 벤더 공식 오픈소스(Apache-2.0)와 검증된 저가 mmWave 모듈 생태계가 성숙했다. 영상 없이 존재·움직임·체류를 신호화할 수 있다는 점이 피팅룸처럼 카메라가 불가능한 공간의 진입 명분이 된다.

**여섯째, K-패션의 빠른 시즌 사이클.** 한국 패션은 6~8 시즌(시즌 2~3개월)이 표준이고 신상품 출시 주기가 2~4주로 짧다. 이게 TryOps의 4주 PoC가 *"한 시즌의 절반"* 으로 비즈니스 의미를 가지는 시점 정당성이다. 서양 SPA의 4 시즌 모델에서는 4주 데이터가 너무 짧지만, 한국에서는 *"신상품 출시 2주차 의사결정"* 의 결정적 윈도우가 된다.

## 4. ICP와 비타겟

### 1차 ICP 조건

다음 조건을 모두 만족하는 브랜드 본사가 초기 타겟이다.

- 직영점 10~50개
- RFID 일부 도입 또는 도입 검토 중
- 피팅룸 의존도가 높은 카테고리 (사이즈·핏 민감도 높음)
- 본사 MD/리테일 운영팀이 존재
- POS/RFID 데이터 추출 가능
- 플래그십·팝업 운영 경험 있음

### 카테고리 우선순위

1. **애슬레저/스포츠웨어 D2C** — 안다르·젝시믹스·뮬라웨어 결의 브랜드
2. **언더웨어/이너웨어** — 비비안·인심내의 결
3. **여성 컨템포러리** — 마뗑킴·마르디 메크르디 결
4. **팝업·플래그십 중심 K-패션 브랜드**

가상 페르소나 브랜드:
- **AURA FIT**: 여성 애슬레저 D2C (안다르·젝시믹스 결)
- **NOVA INNER**: 언더웨어·이너웨어 (비비안 결)
- **MILL STUDIO**: K-컨템포러리 (마뗑킴 결)

### 비타겟

- 보세 로드샵 단독 매장 (데이터 지불 의사 낮음)
- 글로벌 대형 SPA (자체 인프라 보유)
- 단순 방문객 동선 분석 매장 (CCTV analytics 영역)
- 피팅룸 의존도 낮은 카테고리

## 5. 페르소나

세 인물을 정의한다. **구매 의사결정자, 일상 사용자, 데이터 소비자**다.

### 5.1 김지현 (38세) — 본사 데이터팀장 / 구매 의사결정자

**소속**: AURA FIT 본사 데이터팀, 팀장 2년차
**팀 구성**: 데이터 엔지니어 2명, 데이터 분석가 3명
**보고 라인**: COO 또는 CFO
**KPI**: 매장 매출 성장률, 재고 회전율, 신상품 sell-through

**일과 패턴**:
- 오전: 전일 매출 보드 확인, 주간 머천다이징 회의 준비
- 오후: 신상품 출시 4주차 성과 분석, 매장별 재고 이동 의사결정
- 저녁: 분기 임원 보고용 인사이트 작성
- 매일 3~5개 분석 요청이 마케팅·MD·운영팀에서 들어옴

**페인포인트 시간 순서**:

| 시점 | 상황 | 블록 |
|---|---|---|
| 신상품 출시 4주차 | MD팀이 *"린넨 셔츠 sell-through 분석해주세요"* 요청 | *"팔렸나"는 알지만 *"왜 안 팔렸나"는 데이터 없음* |
| 매장 격차 분석 | COO가 *"A매장과 B매장 매출 30% 차이 왜?"* 질문 | 매장별 손님 행동 차이 측정 도구 없음 |
| 가격 인하 회의 | MD가 *"이 SKU 가격 인하할까요?"* 질문 | *"사이즈 문제"* vs *"가격 문제"* 구분 불가, 인하 결정의 근거 부재 |
| 시즌 종료 회고 | 임원이 *"다음 시즌 뭘 다르게 할 건가?"* 질문 | *"데이터 갭"* 때문에 모호한 답변, 본인 팀 신뢰도 손상 |

**TryOps에 끌리는 이유**:
- *"입어봤지만 안 산 이유"* — 본질적 데이터 갭을 메우는 솔루션
- RFID 데이터(이미 보유)와 통합 — 신규 인프라 부담 적음
- 본사 직접 받는 비영상 집계 데이터 — 법적·운영적 부담 없음

**구매 트리거**:
- 시즌 종료 매출 회고에서 *"왜 안 팔렸나"* 답을 못 줘서 임원 추궁 받은 직후
- 신상품 출시 직후 매장 간 격차 큰데 원인 분석 막힐 때
- 가격 인하 vs 사이즈 차트 개선 의사결정 회의에서 의견 안 모일 때
- 경쟁사(Zara, Uniqlo) 분석 사례 본 임원이 *"우리도 비슷한 거 안 하나?"* 라고 물었을 때

**구매 거부감**:
- *"RFID도 이미 활용 못 하고 있는데 또 새로운 데이터 시스템?"*
- 매장 인프라 설치 부담 — 점주 설득 어려움
- 정확도 검증 안 됨 — 파일럿 매장 PoC 필수
- 손님 개인정보 오해 가능성

### 5.2 박서연 (32세) — 머천다이저 / 일상 사용자

**소속**: AURA FIT 본사 MD 8년차 (AURA FIT 3년차)
**담당**: 요가복·필라테스복 라인 (시즌별 80~120 SKU)
**KPI**: 본인 담당 카테고리 매출, 재고 회전율, 신상품 hit-rate

**일과 패턴**:
- 오전 9~11시: 전일 매출 SKU별 확인, 사이즈별 판매 분석
- 11~13시: 매장 매니저들과 메신저로 현장 피드백 수집
- 13~15시: 디자이너·생산팀과 다음 시즌 회의
- 15~18시: 재고 이동 의사결정, 매장별 진열 가이드 수정
- 본인 담당 SKU 80개 × 매장 30개 = 매일 2,400개 데이터 검토

**페인포인트**:

1. **매장 매니저 피드백의 신뢰도 한계** — 매장 A: *"사이즈 안 맞다는 분 많아요"* vs 매장 B: *"디자인을 안 좋아하시는 거 같아요"*. 같은 SKU에 매장마다 다른 정성 피드백. 어느 게 진짜 원인인가?
2. **A/B 테스트 불가** — 핏 수정 효과인지 시즌 효과인지 구분 안 됨
3. **"왜 안 팔리는가"에 대한 무력감** — sell-through 30% (평균 45% 미달), 디자인/가격/핏/진열/마케팅 다 의심 가능, 좁힐 수 없음
4. **가격 인하의 두려움** — 인하하면 브랜드 가치 손상, 안 하면 재고 부담. *"이 SKU는 핏 문제라 인하해도 안 팔린다"* vs *"가격만 낮추면 잘 팔린다"* 구분 불가

**TryOps에 끌리는 이유**:
- Hesitation Score로 *"이 SKU 입어본 사람의 망설임 점수 높음 → 핏 문제 가능성"* 진단
- 사이즈 비교 행동: *"M 들어가서 L 들고 다시 들어왔다 = 사이즈 차트 불일치"*
- 매장 간 비교: 같은 SKU의 hesitation 분포로 매장 환경 문제 진단

### 5.3 정민호 (29세) — 매장 매니저 / 데이터 소비자

**소속**: AURA FIT 강남점 매니저 2년차
**담당**: 강남점 (피팅룸 6개, 스태프 8명)
**KPI**: 매장 일·주·월 매출, 전환율, 평균 객단가

**페인포인트**:

1. **피크 시간 피팅룸 부족 판단** — *"피팅룸 더 만들어야 하나, 회전 빨리 시키면 되나?"* 의 판단 근거 부재
2. **직원 배치 최적화** — 8명 중 누구를 피팅룸 응대 전담? 본사가 인력 충원 거부 (*"본사 데이터로는 인원 충분"*)
3. **본사 일관 정책 답답함** — *"우리 매장은 손님 행동이 다른데 본사가 안 들어줌"*, 매장별 특수성 어필 수단 없음
4. **디자이너 응대 타이밍** — 손님 입실 5분/7분/10분 중 언제? 경험 의존, 신입 못 잡음

**TryOps에 끌리는 이유**:
- 실시간 알람: *"3번 피팅룸 7분 경과 + 활동 강도 낮음 → 직원 개입 권장"*
- 매장 운영 데이터로 본사에 *"우리 매장 토요일 3시 피크가 다른 매장보다 30% 강함"* 실증 어필
- 신입 교육: *"이 타이밍에 응대하면 구매 전환 +20%"* 를 데이터로 가르침
- 컴플레인 사실 확인 30초

### 5.4 세 페르소나의 상호작용

같은 try-on session 데이터가 세 사람에게 다르게 보인다:
- **김지현**: 100매장 집계 → 전사 트렌드 인사이트
- **박서연**: SKU별 집계 → 상품 의사결정 근거
- **정민호**: 매장 실시간 → 운영 액션

이게 **멀티테넌트 + 멀티롤 IAM이 필요한 비즈니스 근거**다. 플랫폼 엔지니어 관점에서 권한 설계의 기반이 된다.

## 6. 의사결정 회의 매핑

데이터는 *"보여주기 위해"* 존재하지 않는다. 어떤 회의에서 어떤 결정을 바꾸는지가 본질이다.

### 6.1 신상품 출시 2주차 리뷰 회의

**질문**: 팔리지 않는 상품은 관심 자체가 없는가, 입어보고 안 사는가? 사이즈별 문제가 있는가?

**TryOps 데이터**: 상품별 try-on count, try-on to purchase rate, 사이즈별 hesitation score, 평균 fitting duration

**바뀌는 의사결정**: 가격 인하 전 사이즈 가이드 개선, 특정 사이즈 재고/패턴 점검, 직원 응대 스크립트 수정

### 6.2 주간 MD / 리테일 운영 회의

**질문**: 이번 주 어떤 상품이 매장에서 막히고 있는가? 매장별 전환 차이의 원인은?

**TryOps 데이터**: 매장별 SKU conversion gap, hesitation distribution, fitting friction score, assistance need 신호 발생률

**바뀌는 의사결정**: A매장 best practice를 B매장에 적용, 특정 상품군의 사이즈 안내 보강, 피크 시간대 직원 배치

### 6.3 가격 인하 / 프로모션 결정 회의

**질문**: 이 상품을 할인해야 하는가, 가격 문제가 아니라 핏/사이즈/소재 문제인가?

**TryOps 데이터**: 피팅룸 반입 높지만 구매 낮음, hesitation 높음, 사이즈별 편차, 대기/응대 신호와 함께 관측되는 실패

**바뀌는 의사결정**: 무작정 할인 대신 사이즈 가이드·피팅 안내·직원 추천 방식 먼저 수정 → 마진 방어

### 6.4 시즌 종료 회고

**질문**: 다음 시즌에 무엇을 바꿔야 하는가? 실패 상품은 왜 실패했는가?

**TryOps 데이터**:
- 관심 부족 상품: try-on 자체 낮음
- 실착 후 실패 상품: try-on 높지만 purchase 낮음
- 사이즈 마찰 상품: 특정 사이즈 hesitation 높음
- 매장 운영 이슈 상품: 특정 매장만 conversion 낮음

**바뀌는 의사결정**: 디자인/마케팅 문제와 핏/사이즈 문제를 분리, 다음 시즌 패턴·사이즈표·상품 설명 개선

### 6.5 매장 운영 개선 회의

**질문**: 피팅룸 대기 때문에 전환 떨어지는 시간대가 있는가? 직원 개입 필요 타이밍은?

**TryOps 데이터**: 토요일 15~18시 fitting friction 상승, 같은 시간대 conversion 하락, assistance signal 발생 후 직원 개입 없는 세션의 구매율 하락

**바뀌는 의사결정**: 피크 시간대 피팅룸 전담 직원 배치, 사이즈 요청 프로세스 개선

## 7. 신뢰 설계 — 고객과 직원이 어떻게 받아들이는가

TryOps는 피팅룸이라는 민감한 공간을 다루므로 제품 신뢰 설계가 기능보다 앞서야 한다.

**v3.4 결정적 발견 — 데이터 맥락 경고 추가**: 오픈서베이 *"개인정보 보호 트렌드 리포트 2024"* 의 *"오프라인 환경 개인정보 불안 76.2%"* 는 정확히는 *"개인정보 기재가 필요한 오프라인 이벤트"* 조건. 매장 비영상 센서는 *"기재 없는 추적"* 이므로 76.2%가 그대로 적용된다고 단정 불가. **방향성으로는 *"민감 다수 가능성"* 시사하지만 정량 검증은 PoC 자체 조사 필수** (상세는 `personas_extended.md` v4 섹션 5.1 참조). 보수적 접근으로 *"다수 일상 대응"* 의 신뢰 설계 기조를 유지한다.

### 7.1 고객 관점

**v3.4 패치 — 다수 일상 대응 구조**: 고객 거부감은 *"소수의 위기"* 가 아니라 *"다수의 일상 우려"* 로 가정. 본 섹션의 표현·고지문·대응 매뉴얼은 모두 보수적 *"다수 응대"* 기조.

**고객이 느낄 수 있는 우려**:
- *"피팅룸 안에서 뭔가 감지한다고?"*
- *"카메라는 아니라지만 찝찝한데?"*
- *"내가 옷 갈아입는 행동이 분석되는 건가?"*
- *"동의 안 받고 강제 수집 아닌가?"* (v3.4 추가 — 다수 우려)
- *"내 데이터 빼달라 요청 가능한가?"* (v3.4 추가)

**피해야 할 표현**:
- "고객의 피팅 행동을 분석합니다"
- "피팅룸 내부 움직임을 추적합니다"
- "동행자 수를 감지합니다"
- "고객별 행동을 기록합니다"

**사용해야 할 표현**:
- "피팅룸 내부를 촬영하지 않습니다"
- "음성·영상·얼굴·신체 이미지를 수집하지 않습니다"
- "개인을 식별하지 않는 익명 신호만 사용합니다"
- "상품 경험 개선을 위한 집계 지표로만 활용합니다"
- "거부 의사가 있으시면 매장 매니저에게 알려주세요" (v3.4 추가 — 거부 권한 명시)

**매장 고지문 v2 (v3.4 — 다수 응대 구조)**:
> 본 매장은 카메라·녹음·개인 식별 없이 매장 혼잡도만 익명으로 측정합니다. 거부 원하시면 매장 매니저에게 알려주세요.

**매장 직원 5개 표준 질문 답변 (v3.4 신규 — 다수 응대 필수)**:

손님 1명이 질문하면 직원이 즉시 답할 수 있는 5개 표준 질문 매뉴얼. 다수가 일상적으로 질문할 가능성이 있으므로 *"전 직원 표준 답변 교육"* 이 신뢰 설계의 핵심.

- Q1 *"뭘 측정하나요?"* → "혼잡도, 평균 체류시간 (개인 식별 없음)"
- Q2 *"카메라인가요?"* → "아닙니다. 영상·음성 일절 없습니다"
- Q3 *"개인 데이터 빼달라 요청 가능한가요?"* → "매장 단위 익명이라 개인 식별이 안 됩니다만, 거부 원하시면 매니저에게 알려주세요"
- Q4 *"호흡 측정한다는데?"* → "심박/호흡은 측정하지 않습니다. 단순 혼잡도만"
- Q5 *"동의받았나요?"* → "매장 입구 안내문으로 통지하고 있습니다. 추가 동의 원하시면 알려주세요"

**SNS 위기 대응 매뉴얼 (v3.4 신규)**:

다수 거부감이 SNS로 가는 임계점 모니터링 필수.
- 본사·매장·TryOps 3자 24시간 대응팀
- 도입 전 IT 매체·소비자단체 사전 커뮤니케이션
- 위기 발생 시 24시간 이내 공식 답변

상세 손님 페르소나 분석은 `personas_extended.md` v4 섹션 5·6 참조.

### 7.2 직원 관점

**v3.4 패치 — 매장 직원의 본사 정책 영향력**: 한국경제 *"백화점 VOC 3회면 본사 계약 불이익을 매장 직원이 정확히 알고 활용"* 사례 검증 (`personas_extended.md` v4 섹션 3 참조). 매장 직원이 본사 정책에 영향력을 행사할 수 있다는 사실은 신뢰 설계에 두 가지 함의:

1. **직원이 *"매장 단위 거부"* 압력을 본사에 가할 수 있음** — 직원 우호도가 도입 성패의 핵심
2. **직원 본인의 *"방패 데이터"* 가 필요** — 본사 압박에서 매장을 방어할 수 있는 매니저 본인 대시보드 제공

**직원 우려**:
- *"이거 우리 감시하는 거 아니야?"*
- *"직원 개입 안 했다고 평가에 쓰는 거 아니야?"*
- *"매장별/직원별 압박 지표가 되는 거 아니야?"*
- *"또 본사가 일 늘리는 거 아니야?"* (v3.4 추가 — Accenture 매장 직원 7/8 불만족, McKinsey 유연성 부족 1순위 퇴사)

**좋은 범위** (제공함): 매장별 피팅룸 병목, 시간대별 대기 강도, SKU별 hesitation, 직원 개입 필요 후보 알림, **매니저 본인 대시보드** (v3.4 추가 — 본사 압박 방어용)

**위험한 범위** (제공하지 않음): 직원별 응대 점수, 직원별 전환율 랭킹, 고객별 상세 행동 기록, 실시간 감시 화면

**제품 포지셔닝**:
> 직원 감시가 아니라, 피팅룸 병목과 고객 응대 타이밍을 줄이는 운영 보조 도구.

### 8.2 중대한 인사이트 — RFID 도입 현황의 실증 확인

본 v3.1 시장 분석 결과 **한국 패션 RFID 도입 확인 기업 5곳**이 식별됐다 (출처: 패션포스트, 한국경제, 한세엠케이 인터뷰 기사 등):

| 기업/브랜드 | RFID 상태 | 확인 출처 |
|---|---|---|
| 삼성물산 패션 (에잇세컨즈) | 전 매장 도입 완료 | 패션포스트 *"비용 절감 솔루션 RFID"* |
| 코오롱인더스트리 FnC | 도입 진행 | 패션포스트 |
| 에프앤에프 (디스커버리·MLB) | 도입 진행 | 패션포스트 |
| 크리스에프앤씨 (골프웨어) | 도입 운영 중 (성공 사례) | 패션포스트, 2012년 정부 지원 후 유일 유지 |
| 한세엠케이 (NBA·TBJ) | 도입 운영 중 | 한국경제 2019.04 인터뷰 |

추가로 *"국내 패션 업체 중 RFID를 도입한 업체는 10개 안팎 수준, 2012년 정부 지원 후 대부분 롤백"* (출처: 패션포스트). 

이 사실이 비즈니스에 미치는 영향:
- **Funnel Module의 진짜 영업 가능 기업은 5~10개로 매우 좁음**
- **K-D2C 본사(안다르·젝시믹스 등)는 RFID 미확인** — Funnel 직판 어려움
- **Flow Module이 K-D2C 본사 영업의 메인 진입 도구로 격상 필요**
- GTM 사이클: Flow 도입 → 본사 RFID 도입 → Funnel 업그레이드

상세 검증과 향후 검증 과제는 `market_analysis_detail.md` 섹션 2 참조.

### 8.3 Beach-head 본사 — 두 시나리오 분리

본 v3.1 시장 분석에서 결정적 발견: **한국 패션 RFID 도입 확인 기업은 5곳뿐**이고 안다르·젝시믹스·마뗑킴은 RFID 도입 미확인. 따라서 Beach-head를 RFID 도입 여부에 따라 두 시나리오로 분리.

상세 검증은 `market_analysis_detail.md` 섹션 2 참조.

**시나리오 A — RFID 도입 확인 기업 (Funnel Module 직판)**:

### 8.5 "아무것도 안 함"과의 경쟁

가장 강한 경쟁자는 *"아무것도 안 하는 것"* 이다. 고객은 다음과 같이 생각할 수 있다:

> 지금도 매출 보고 있다. 직원 리포트도 있다. RFID도 이미 있다. 굳이 피팅룸 행동 신호까지 봐야 하나?

따라서 TryOps는 **비도입 비용**을 설명해야 한다:

- 가격 문제 아닌 상품을 할인해 마진 잃음
- 사이즈·핏 문제를 리뷰와 반품 이후에야 발견
- 매장별 전환 차이를 감으로만 판단
- 피팅룸 병목으로 피크 시간대 전환율 놓침
- high interest / low conversion 상품을 단순 실패로 오판
- RFID 데이터가 session 단위로 해석되지 못하고 이벤트 로그로 남음

### 8.6 확장 시나리오 (5년)

```
Year 1-2: 애슬레저·언더웨어 Beach-head (안다르·젝시믹스·비비안)
Year 2-3: K-컨템포러리 확장 (마뗑킴·마르디 메크르디)
Year 3-4: 백화점 자체 브랜드 (한섬·신세계인터내셔날 산하)
Year 4-5: 골프웨어·슈즈·인접 카테고리
Year 5+:  비의류 (가구·화장품 매장, Flow 단독)
```

지리적 확장은 **K-패션 글로벌 진출에 묻어가는 전략** — 안다르가 미국 진출하면 우리도 미국 매장에 노드 도입. 안다르·젝시믹스가 우리의 글로벌 진입 발판.

상세 시나리오는 `market_analysis_detail.md` 섹션 6 참조.

## 9. 법적 검증 (한국 법규 기준)

TryOps는 민감한 공간을 다루므로 사전 법적 검증이 필수다. 다음 6개 법령에 대한 검토 결과를 정리한다.

### 9.1 개인정보보호법 — 영상정보처리기기 규정

**핵심 조항**: 제25조 — *"누구든지 불특정 다수가 이용하는 목욕실, 화장실, 발한실, 탈의실 등 개인의 사생활을 현저히 침해할 우려가 있는 장소의 내부를 볼 수 있도록 고정형 영상정보처리기기를 설치·운영하여서는 아니 된다."*

**TryOps 적용**:
- mmWave·WiFi CSI 센서는 **카메라가 아니므로** 영상정보처리기기에 해당 안 함
- 피팅룸 내부 설치 가능 — **법적으로 우리만 살아남는 자리**
- 제25조의 까다로운 규정(안내판, 운영 방침 공개, 영상 보관 기준)에서 면제

### 9.2 개인정보보호법 — 일반 규정

**리스크**: 집계 데이터는 안전하지만 세션 단위 데이터는 RFID/POS와 결합 시 *"다른 정보와 결합하여 알아볼 수 있는 정보"* 로 분류될 가능성.

**대응**:
- 본사로 가는 데이터는 **익명 집계만** 전송
- 매장 raw CSI는 **7일 후 자동 삭제**
- 세션 단위 데이터는 **가명처리** (예약 데이터와 직접 조인 금지)
- 매장 입구 안내문 의무 부착

### 9.3 근로자참여 및 협력증진법 — 근로자 감시 설비

**리스크**: TryOps가 직원 활동을 감시하는 도구로 해석될 가능성. 상시 30인 미만 사업장은 노사협의회 의무 없지만 개인정보 동의는 필수.

**대응**:
- 도입 전 직원 설명회 + 동의서 (수집 정보, 목적, 보관 기간 명시)
- 본인 데이터 열람 권리 보장
- *"감시"가 아닌 *"운영 데이터"* 로 프레이밍
- 초기 제품에서는 **직원별 평가 지표 미제공**

### 9.4 의료기기법 — 가장 위험한 영역

**리스크**: 호흡·심박 측정 기능은 의료기기 인증 대상 가능성. *"응급 감지"*, *"건강 모니터링"* 같은 마케팅은 위험.

**대응**:
- Phase 1에서는 호흡/심박을 **숫자로 표시 안 함**, *"활동 강도 점수"* 로 우회
- 의료적 주장(응급 감지, 건강 모니터링) **절대 금지**
- 명시 고지: *"본 서비스는 의료기기가 아니며 의료적 진단·치료 목적으로 사용할 수 없습니다"*

### 9.5 위치정보법

**리스크 없음**. 위치정보법은 스마트폰 등 통신단말장치 위치를 다룸. TryOps 센서는 공간 신호 변화 측정이므로 적용 안 됨.

**금지선**: 향후 스마트폰 MAC 트래킹 기능 **절대 추가 금지**

### 9.6 전파법 / KC 인증

ESP32-S3 자체는 글로벌 인증, 우리 하우징·펌웨어 조합 상용 출시 시 **KC 인증 필요**. Phase 1·2는 자체 사용 목적이라 면제, Phase 3 상용화 전에 인증 진행.

### 9.7 컴플라이언스 체크리스트

법무팀 검토 시 제공할 체크리스트는 별도 문서(`legal_review.md`)에 정리. 핵심 7개 항목:

1. 영상정보처리기기 해당 여부 (해당 없음 확인)
2. MAC 트래킹 여부 (없음 확인)
3. 호흡/심박 마케팅 회피 (확인)
4. 직원 동의서 (작성됨)
5. 고객 안내문 (작성됨)
6. 데이터 보존/삭제 정책 (명시)
7. KC 인증 (Phase 3 전 예정)

## 10. Joint Signal 5종

### 10.1 Hesitation Score

같은 상품군 또는 같은 SKU의 다른 사이즈가 반복 입출고되는 동안 CSI에서 긴 체류·반복 움직임·거울 앞 정체 후보가 관측되는 정도.

예시:
```
레깅스 M → 2분 후 퇴실
레깅스 L → 7분 체류
레깅스 S → 5분 체류 + 반복 움직임
결과: 구매하지 않음 → 사이즈 차트 신뢰도 문제 후보
```

### 10.2 Assistance Need Signal

입실 후 일정 시간 경과, 추가 사이즈 요청 이벤트 없음, CSI상 움직임 낮고 체류 길어지는 상태 → 고객이 도움 필요하지만 호출 못하는 상태 후보.

### 10.3 Companion Effect

하나의 try-on session 안에서 2명 이상 존재 가능성 또는 동행자 행동 패턴 관측. *"정확한 인원 측정"* 으로 표현 안 함, *"동행 가능성 신호"* 로 톤다운.

### 10.4 Fitting Friction Score

RFID 회전율 + CSI 대기 강도 결합 점수.
- 상황 A: RFID 회전율 높음 + CSI 대기 강도 높음 → 피팅룸 돌아가지만 대기 병목
- 상황 B: RFID 회전율 보통 + CSI 대기 강도 낮음 → 피팅룸 진입 전 포기 (RFID로 절대 못 잡는 신호)

### 10.5 Phantom Try-on Detection

CSI상 입실 감지되었지만 RFID 이벤트 없는 경우. RFID 데이터 품질 검증 도구. 본사 RFID 운영팀에게 직접적 가치.

## 11. 제품 모듈 구조

```
TryOps — 카메라 없는 피팅룸 인텔리전스 플랫폼

├─ Core Platform
│  ESP32-S3 노드 + CSI 처리 + 클라우드 인프라
│
├─ Funnel Intelligence Module  (메인)
│  ├ 대상: RFID 도입 매장
│  ├ 입력: CSI + RFID + POS
│  ├ 분석: Joint Signal 5종
│  └ 가격: 매장당 월 15만~30만원
│
└─ Flow Intelligence Module  (확장)
   ├ 대상: RFID 미도입 매장
   ├ 입력: CSI 단독 (+ POS)
   ├ 분석: 공간 흐름 5종 (점유, 대기, 체류, 회전, 이상상태)
   └ 가격: 매장당 월 5만~10만원
```

Funnel Module이 메인 수익원이지만, **Flow Module은 단순 확장 모듈이 아니라 K-D2C 본사 영업의 메인 진입 도구**다.

본 v3.1 시장 분석 결과 한국 패션 RFID 도입 기업은 약 5~10개에 불과(섹션 8.2 참조)하고, 주요 K-D2C 브랜드(안다르·젝시믹스·마뗑킴)는 RFID 도입 미확인. 따라서 GTM 사이클은 다음과 같이 설계된다:

```
K-D2C 본사 영업 → Flow Module 도입 (RFID 없이 즉시 가능)
       ↓
본사가 RFID 도입 결정 (우리 권유 또는 자체 결정)
       ↓
Funnel Module 업그레이드 (RFID + CSI 통합)
       ↓
본사 ARPU 매장당 5만원 → 15만원 + 자연 인상
```

이 사이클이 사업 핵심. Flow가 *"진입 훅"* 이상의 전략적 무기다.

두 모듈은 **같은 Core Platform 위에서 분기**되어 코드 공유율 약 70%.

상세 모듈 차이는 별도 문서(`docs/04_modules/funnel_vs_flow.md`) 참조.

## 12. 가격 정책과 Unit Economics

### 12.1 가격 구조

| 플랜 | 대상 | 가격 모델 | 매장당 환산 |
|---|---|---|---|
| Flow Standalone | RFID 미도입 매장 | 매장당 월 5만원 | 5만원 |
| Funnel Starter | 20매장 이하 본사 | 매장당 월 9만원 | 9만원 |
| Funnel Pro (K-D2C 본사) | 21~100매장 본사 | 매장당 월 15만원 | 15만원 |
| **Funnel Enterprise ARR (대기업)** | **100매장 이상 RFID 도입 대기업** | **본사 연 ARR 패키지** (1.5억~3억원/년) | **8~12만원** |

**일회성 도입비**: 매장당 약 30만원 (노드 4~6개 + 게이트웨이 + 설치)

**ARR 패키지 도입 이유**: 박정우 페르소나(대기업 데이터팀장, 상세 `personas_extended.md` 참조) 분석 결과, 대기업 결재는 *"매장당 단가"* 가 아니라 *"연 ARR 패키지"* 가 자연스러움. 6~8단계 결재 프로세스를 통과하려면 단가 협상이 아닌 ARR 일괄 계약이 영업 효율적.

🔴 **단, 본 결론의 근거는 일반론 + 박정우 페르소나 행동 검증 30% 수준**. PoC 단계 대기업 본사 실제 영업 시 검증 필수. 한국 패션 대기업이 *"본사 연 ARR 1.5억~3억 일괄 계약"* 을 즉시 채택할지는 미검증.

### 12.2 Unit Economics — 가격 근거

매장당 한계 비용 (cloud + 운영):
- Kinesis on-demand: 약 1만원
- S3 + Athena: 약 2만원
- 매장 게이트웨이 (감가 1/60): 약 0.5만원
- 노드 6개 (감가 1/60): 약 1만원
- **소계: 4.5만원/매장/월**

매장당 인건비 분산 (영업/CS, 100매장 운영 시): 약 3만원/매장

**총 한계 비용**: 약 7.5만원/매장/월

목표 마진 60% → 가격 ≈ 7.5 / 0.4 ≈ **18.75만원**

**플랜별 마진율 (한계비용 7.5만원/매장/월 기준)**:

| 플랜 | 매장당 환산 | 마진 (절대) | 마진율 | 의미 |
|---|---|---|---|---|
| Flow Standalone | 5만원 | -2.5만원 | 손실 | **단독 수익성 없음**, Funnel 진입 훅 또는 RFID 도입 유도용 |
| Funnel Starter | 9만원 | 1.5만원 | 16.7% | **신규 본사 진입용 미끼 가격**, 마진 손해 감수 |
| Funnel Pro | 15만원 | 7.5만원 | 50% | 메인 수익원 (K-D2C 본사) |
| Funnel Enterprise ARR | 8~12만원 환산 | 0.5~4.5만원 | 6~38% | 대기업 ARR 패키지, 본사 ARR 1.5억~3억원로 영업 효율 보상 |

**Unit Economics 함의**:
- Starter는 신규 본사 락인용 도구, 첫 6~12개월 손실 감수
- 본사가 Starter에서 Pro로 업그레이드해야 사업성 확보
- Flow Standalone은 단독 사업 아님 — 시장 침투 또는 Funnel 진입 경로로만 활용
- Enterprise ARR는 매장당 단가 낮지만 본사 연간 1.5억~3억 일괄 계약으로 영업 효율 보상
- 본사당 평균 ARPU가 매장당 12만원 이상 유지되어야 전체 사업성 성립

### 12.3 가격 방어 논리

- 전환율 +1~2%p 개선 가능성으로 월 구독료 방어 가능
- 가격 인하 회피로 마진 방어 가능
- 신상품 출시 초기 실패 원인 후보 탐지로 다음 시즌 개선

## 13. ROI 모델

### 13.1 전환율 개선 ROI

기본 공식:
```
추가 매출 = 월간 try-on 세션 수 × 개선 가능한 전환율 상승폭
         × 평균 객단가 × 기여율
```

예시 가정 (각 가정의 출처 명시):

| 변수 | 값 | 출처 / 가정 |
|---|---|---|
| 월간 try-on 세션 수 | 3,000건 | 매장 일평균 100건 × 30일, PoC 검증 필요 |
| 현재 try-on purchase rate | 25% | 패션 리테일 평균 추정, PoC 측정 필요 |
| 개선 가능한 상승폭 | +1.5%p | RFID 도입 사례 기반 보수 추정, PoC 검증 필요 |
| 평균 객단가 | 70,000원 | 애슬레저 카테고리 평균 추정 |
| 기여율 | 50% | TryOps 외 변수 영향 절반 가정 (보수적) |

계산:
```
3,000 × 0.015 × 70,000 × 0.5 = 1,575,000원 / 월 기여 매출
```

→ 매장당 월 15만~30만원 구독료 방어 가능.

**중요**: 위 모든 가정은 PoC 단계에서 정밀 검증 필요. 현재는 가정 기반 추정.

### 13.2 마진 방어 ROI

매출 증가뿐 아니라 **불필요한 가격 인하 회피** 효과:
- 가격 문제 아닌데 할인하는 실수 감소
- 할인 전 다른 액션 테스트
- 시즌 중 상품 개선 우선순위 결정

### 13.3 상품기획 ROI

장기 효과:
- 신상품 출시 초기 실패 원인 후보 발견
- 사이즈별 구매저항 확인
- 반품·리뷰 발생 전 조기 신호 확보
- 매장 직원 정성 코멘트의 정량 보완

## 14. MVP와 PoC

### 14.1 MVP — 내부 개발 범위

**가장 먼저 증명할 가설**:
> Hesitation Score는 RFID-only funnel보다 high try-on / low conversion 상품의 원인 후보를 더 잘 구분할 수 있다.

**왜 Hesitation Score부터인가**:
- MD/상품기획팀이 바로 이해함
- *"왜 안 샀는가"* 의 제품 핵심 가치와 직결
- RFID-only와의 차별점이 가장 선명
- 시뮬레이션 데이터로 MVP 구현 가능
- 포트폴리오 시각화 임팩트 큼

**보조 가설**: Phantom Try-on Detection
- 정확도 검증이 즉시 가능 (RFID 누락 여부 즉시 확인)
- 본사 RFID 운영팀 즉시 가치 인식
- 영업 훅: *"PoC 첫 주에 Phantom 5건 발견"*

**MVP에서 하지 않을 것**:
- 정확한 행동 인식 주장
- 고객 개인 식별
- 직원별 감시 지표
- 실시간 CCTV식 모니터링
- 모든 Joint Signal 동시 구현
- 완전 자동 POS/RFID 연동

### 14.2 PoC — 고객 검증 패키지

**PoC의 위치 명확화**: 4주 PoC는 **영업 사이클의 진입 도구가 아니라 본사 내 확장 의사결정 도구**다. 시장 분석(`market_analysis_detail.md` 섹션 6) 결과 한국 패션 본사 영업 사이클은 다음과 같다:

| 시나리오 | 영업 사이클 총 기간 | PoC 단계 위치 |
|---|---|---|
| RFID 도입 대기업 (Funnel) | 8~13개월 | 6~8개월차 |
| K-D2C 본사 (Flow) | 6~9개월 | 4~5개월차 |

PoC는 본사를 *"확보하는"* 도구가 아니라, 이미 검토 단계에 들어온 본사가 *"3개 매장에서 확장 의사결정"* 하는 단계의 도구다.

**4주 PoC 구성**:

| 주차 | 작업 | 성공 기준 |
|---|---|---|
| Week 1 | 데이터 수집 안정화, 베이스라인 측정 | CSI/RFID/POS 통합 적재 안정 |
| Week 2 | 첫 try-on session 마트 생성 | session matching 정확도 90%+ |
| Week 3 | high try-on / low conversion 후보 발굴 | 5개 이상 SKU 후보 도출 |
| Week 4 | MD 회의에서 액션 1개 채택 + 결과 보고 | 실제 운영 액션 1개 이상 채택 |

**4주에서 검증 가능한 것**:
- Hesitation Score와 conversion의 관계
- RFID-only로는 안 보이고 TryOps로는 보이는 케이스
- 특정 사이즈/매장/시간대 반복 패턴

**4주에서 검증 불가능한 것**:
- 시즌 전체 효과 (8주 이상 필요)
- A/B 테스트 효과 (별도 실험 설계 필요)
- 매출 증가 인과관계 (12주 이상)

**PoC vs Pilot 구분**:
- PoC: 4주, 직영점 3개, 가설 검증
- Pilot: 12주, 직영점 10~20개, 효과 측정 + 확장 의사결정

### 14.3 PoC 패키지

```
기간: 4주
대상: 직영점 3개
범위: 상위 50개 SKU 또는 특정 신상품 라인
데이터: RFID 이벤트, POS 이벤트, CSI activity window
제공물: 주간 리포트, 최종 PoC 리포트, action candidate 리스트
핵심 검증: Hesitation Score가 RFID-only 리포트보다 더 나은 원인 후보를 제공하는가
```

고객 제공: 매장 정보, RFID/POS export, SKU master, 운영 캘린더, 가격 변경 이력

TryOps 제공: 센서 설치 또는 시뮬레이션 기반 분석 환경, try-on session reconstruction, high try-on/low conversion 리포트, hesitation score 리포트, phantom 후보 리포트, 운영 액션 후보, PoC 결과 요약

## 15. 실패 시나리오

TryOps는 강점이 있지만 실패 가능성도 크다. 다음 6가지 실패 시나리오와 대응을 미리 정의한다.

### 실패 1: CSI 신호가 실제 매장 환경에서 불안정

피팅룸의 벽·거울·금속 행거·사람 밀집·매장 WiFi 간섭 때문에 hesitation 안정적 구분 실패 가능.

**대응**: *"정확한 행동 인식"* 주장 안 함, hesitation을 *"행동 강도 기반 후보 지표"* 로 표현, RFID/POS 상관관계로 유효성 검증, 매장별 calibration 필요성을 제품 설계에 포함.

### 실패 2: 고객 프라이버시 거부감 — 다수 일상 거부 + SNS 위기

**v3.4 패치 — 위험도 격상**: 손님 페르소나 v4 검증 결과 *"오프라인 환경 개인정보 불안 76.2%"* (오픈서베이 2024, 맥락 경고 적용). 매장 비영상 센서의 진짜 거부감 비율은 미검증이지만 *"소수 위기"* 보다 *"다수 일상 거부"* 일 가능성이 더 크다.

세 가지 실패 경로:

1. **다수 일상 거부 → 매장 도입 자체 무산**: 매장 손님 다수가 안내문에 거부 의사 표명 → 매장 매니저가 본사에 도입 철회 요청 → 사업 자체 불가능
2. **SNS 위기 단일 사건 → 본사 즉시 도입 철회**: 쿠팡 개인정보 유출 사례(2025)처럼 한국 사회의 개인정보 위기 민감도 극도로 높음. 매장 비콘 SNS 논란 1건이 본사 압박 야기
3. **언론·소비자단체 사전 공격 → 도입 자체 막힘**: 인스타그램 위치정보 강제 동의 한국 논란 사례처럼 도입 전 사전 공격 가능성

**대응 (v3.4 격상 — 다수 응대 구조)**:
- 매장 안내문은 *"기술 용어"* 가 아니라 *"고객 안심 메시지"* + *"거부 권한 명시"*
- 매장 직원 5개 표준 질문 답변 매뉴얼화 (다수 손님 응대 필수)
- 본사·매장·TryOps 3자 24시간 SNS 모니터링 + 위기 대응팀
- 도입 전 IT 매체·소비자단체 사전 커뮤니케이션 (선제 신뢰)
- PoC 매장 손님 인터뷰 30~50건으로 진짜 거부감 비율 측정 (오픈서베이 76.2% 맥락 경고 적용)
- 영상/음성/개인식별 없음, 원천 CSI 7일 후 삭제, 본사 익명 집계만

상세는 `personas_extended.md` v4 섹션 5·6 + `legal_review.md` (예정) 참조.

### 실패 3: RFID/POS 연동 난이도

브랜드마다 POS/RFID 시스템 달라서 연동 비용 큼.

**대응**: Phase 1은 CSV/Parquet batch upload 기반, Phase 2 webhook/API 연동, vendor별 adapter 구조, 완전 자동 연동보다 파일 업로드 MVP 우선.

### 실패 4: 본사가 데이터 봐도 액션 안 함 + 거부형 임원 차단

**v3.4 패치 — 거부형 임원 정밀화**: 페르소나 v4 검증 결과 두 종류의 거부형 임원이 사업을 막을 수 있다 (`personas_extended.md` v4 섹션 2·4 참조):

- **김상철 (시나리오 A 거부형)**: 데이터 안 믿는 본부장, *"매장 인테리어 리뉴얼 vs TryOps"* 결재 차단. HBR Korea 변화 4분의 3 실패율 + 한국 RFID 4분의 3 롤백 트라우마.
- **이수정 (시나리오 B 거부형)**: K-D2C 마케팅 본부장, *"이미 에코마케팅 쓰고 있는데..."* 부서 예산 충돌 거부. 안다르의 에코마케팅 의존 사례로 패턴 검증.

좋은 리포트도 MD/운영팀이 액션 안 하면 가치 없음. 거부형 임원이 결재 단계에서 차단하면 가치 자체 발생 불가.

**대응**:
- 리포트가 아니라 액션 후보까지 제공, 각 액션 담당자·상태 관리, 시즌 회고·주간 MD 회의 포맷에 맞춤, *"관측 → 원인 후보 → 권장 액션 → 결과 추적"* 구조 유지
- **거부형 임원 우회 전략 (v3.4 신규)**: 김상철 같은 본부장 → 박정우(데이터팀장)를 통한 우회 영업 + 동료 본부장 사례 + 현장 매니저 후기. 이수정 같은 마케팅 본부장 → *"매장 데이터, 마케팅 영역 무관"* 차별화 메시지
- **거부형 시간 분해**: 영업 사이클 8~13개월(시나리오 A) / 6~9개월(시나리오 B) 중 절반은 거부형 설득 시간 (상세 `personas_extended.md` v4 섹션 8)

### 실패 5: RFID-only로 충분하다는 판단

고객 질문: *"RFID만으로도 들어가고 나온 거 알 수 있는데 굳이 CSI?"*

**대응**:
- RFID는 무엇이 들어갔는지
- CSI는 그 안에서 망설였는지, 반복 비교했는지, 대기·마찰 있었는지
- POS는 결국 샀는지
- TryOps는 RFID 대체가 아니라 **해석력 보강 레이어**

### 실패 6: ROI가 충분히 크지 않음

전환율 개선폭 작으면 구독료 정당화 어려움.

**대응**: 피팅룸 의존도 높은 카테고리로 제한, 언더웨어·스포츠웨어·애슬레저 우선, 상위 SKU 중심 분석, 4주 안에 high try-on/low conversion 후보 도출, 전환율 아니어도 마진 방어·상품기획 가치 제시.

### 실패 7: RFID 도입 정체 — 시장 리스크

검색 검증 결과 *"한국 패션 RFID 도입 약 10개 안팎, 정부 지원 종료 후 도입 정체, 도입 후 롤백 사례 다수"*. Funnel Module 시장이 정체될 가능성.

**대응**: Flow Module을 시장 침투 도구로 활용 (섹션 8.2, 11 참조). RFID 없는 매장도 우리 고객으로 만들 수 있다는 점이 사업 안전판.

### 실패 8: 기존 RFID 솔루션사의 행동 분석 진입 — 경쟁 리스크

셀메이트·idro 같은 기존 RFID 솔루션사가 *"피팅룸 행동 분석 모듈"* 을 추가 출시하면 차별점 약화.

**대응**:
- 첫 1~2년 안에 Beach-head 본사 5곳 잠금
- 센서 융합·EPC 조인 레이어의 기술 우위 (자체 IP) + Ghost-read 필터 등 RFID 데이터 품질 모듈
- 솔루션사와 파트너십 (그들의 RFID 데이터를 우리가 받아 분석)

### 실패 9: 본사 의사결정 보수성 — 도입 리스크

한국 패션 본사는 데이터 도구 도입에 매우 보수적. *"RFID도 활용 못 한다"* 는 현실이 *"또 다른 분석 도구"* 도입에 부정적으로 작용할 수 있다.

**대응**:
- 4주 PoC로 빠른 가치 증명
- *"RFID 도입 ROI를 살리는 도구"* 메시지로 기존 투자 회수 명분 제공
- 본사 데이터팀의 1년차 KPI에 추가 가능한 형태로 패키징

### 15.10 거부형 페르소나의 실패 시나리오 매핑

위 9가지 실패 시나리오와 거부형 페르소나 3인의 영향을 정리. 상세 페르소나 분석은 `personas_extended.md` v4 섹션 2·3·4 참조.

| 거부형 페르소나 | 영향 실패 시나리오 | 차단 단계 | 우회 전략 |
|---|---|---|---|
| **김상철** (52, 본부장) | 실패 4 (액션 안 함), 실패 9 (본사 보수성) | 결재 단계 (3~6개월차) | 박정우(데이터팀장) 챔피언화 → 본부장 우회. 동료 본부장 사례 + 6개월 ROI 시뮬레이션 |
| **강희정** (51, 시니어 매장 매니저) | 실패 2 (프라이버시 거부감) | 매장 단위 도입 단계 (6~8개월차) | 시범 매장 선정 시 회피. 노드 설치 1시간·매장 작업 0 약속. 매니저 본인 대시보드 제공 (방패 데이터) |
| **이수정** (44, K-D2C 마케팅 본부장) | 실패 4 (액션 안 함, 시나리오 B) | 부서 협의 단계 (1~2개월차) | TryOps는 *"마케팅 영역 무관"* 차별화. 데이터팀 예산으로 추진 (마케팅 예산 분리). 김지현 단독 결재 가능성 활용 |

**영업 사이클별 거부형 페르소나의 실제 활용**:
- 시나리오 A 영업 시간표(8~13개월) 중 **3~6개월차 김상철 설득**, **6~8개월차 강희정 회피**가 핵심 변곡점
- 시나리오 B 영업 시간표(6~9개월) 중 **1~2개월차 이수정 우회**가 핵심 변곡점

거부형 페르소나는 *"우려할 인물의 리스트"* 가 아니라 **영업 사이클 시간 분해의 시간 축 결정자**. 페르소나 분석이 GTM 계획의 자원 배분(영업 시간·예산·인력)으로 직결되는 사례.

## 16. 학습 로드맵

기능 추가 순서가 아니라 학습 순서로 짠다.

### Phase 0: 문제 검증 (현재 단계, 약 2주)

**제약 인식**: 본 프로젝트는 포트폴리오 단계이므로 실제 브랜드 본사 인터뷰는 진행하지 않는다. 인터뷰 기반 검증은 향후 Discovery 과제로 분리한다.

**Phase 0 검증 방식**:
- 공개 자료 기반 시장·경쟁사 검증 (브랜드 IR, 채용 공고, 공개 인터뷰, 통계청 자료)
- 한국 법규 자체 검토 (개인정보보호법, 의료기기법 등 6개 법령)
- 합성 데이터 시나리오 설계

**Phase 0 산출물**:
- 본 전략 문서 (product_strategy.md)
- 보조 문서 6~10종 (lessons_learned, deep_dive, legal_review 등)
- 시뮬레이터 명세
- 데이터 모델 ERD

**Phase 0 검증 한계**:
- 본사 페르소나의 실제 페인포인트 검증 불가 → Phase 1·2에서 합성 데이터 기반 알고리즘 검증으로 대체
- 실제 매장 환경 변수 검증 불가 → Phase 2에서 노드 실측으로 부분 검증
- 시장 규모·가격 민감도 정밀 검증 불가 → PoC 단계 과제로 분리

**본 포트폴리오 단계의 명시적 한계**: 본 프로젝트는 포트폴리오 단계에서 검증을 멈춘다. 실제 본사 인터뷰·매장 손님 인터뷰는 *"실제 사업 시작 시 사용할 골격"* 으로 `personas_extended.md` 섹션 12.5 인터뷰 액션 플랜에 명시 분리. 인터뷰 56~91건 약 500만원 예산은 **본 포트폴리오 단계에서 진행 안 함** — 실제 사업 시 1년차에 진행할 예정.

### Phase 1: 데이터 MVP (4~6주)
- 시뮬레이션 RFID/POS/CSI 이벤트 생성기
- try-on session reconstruction ETL
- Hesitation Score 산출 알고리즘
- high try-on / low conversion 대시보드
- 100매장 합성 데이터로 분석 마트 검증

### Phase 2: Pilot MVP (8~12주)
- ESP32-S3 노드 실측 데이터 (본인 환경)
- 합성 데이터의 현실 보정
- batch upload 기반 POS/RFID 연동
- 주간 MD 리포트 자동 생성
- 모듈 분리 (Funnel / Flow)

### Phase 3: Platform 확장 (12주+)
- vendor별 adapter
- 다매장 비교 분석
- 권한·멀티테넌트 구조
- 장기 데이터 mart
- **Polars → Spark 마이그레이션 (매장 200개 임계점)**

### Phase 4: Advanced Signal (장기)
- Assistance Need 실시간 알람
- Fitting Friction 상세
- Phantom Try-on 운영 통합
- ML feature store
- 온라인 funnel 통합

## 17. 최종 포지셔닝

**TryOps는 RFID가 알려주는 '무엇을 입어봤는가'와 POS가 알려주는 '무엇을 샀는가' 사이의 공백을 mmWave·WiFi CSI 기반 비영상 활동 신호로 보완해, K-D2C 브랜드 본사가 high try-on / low conversion 상품의 원인 후보를 발견하고 상품·사이즈·피팅룸 운영 액션으로 연결하도록 돕는 오프라인 리테일 Conversion Intelligence 플랫폼이다.**
